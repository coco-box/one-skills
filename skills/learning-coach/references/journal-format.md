# 学习档案规范

## 目录结构

```text
learning/
├── PROGRESS.md
├── plan.md
├── sessions/
├── artifacts/
└── templates/
```

如果目录尚不存在且用户明确要求开始留痕，可创建最小结构；不要为了形式创建空目录或空模板。

## 会话记录

文件名使用 `sessions/YYYY-MM-DD-NNN-slug.md`。编号优先读取 `PROGRESS.md` 的最后一条记录后递增；没有历史记录时从 `001` 开始。

每份记录应包含：

1. 背景与本次目标；
2. 关键结论；
3. 决策、假设与约定；
4. 产出物及仓库相对路径；
5. 待办复选框；
6. 与计划、进度和前次会话的关联。

仅保留确实帮助理解学习决策的少量关键原话。摘要必须脱离聊天上下文仍可理解。

## 进度文件

`PROGRESS.md` 至少维护：当前阶段、阶段进度、里程碑、会话索引、待办和薄弱点。每次留痕确保会话待办与总进度一致，并为已完成里程碑附上证据路径。

只有目标、阶段或时间预算发生实质变化时才更新 `plan.md`，并在会话记录中说明原因。

## 产出归档

按阶段存入 `artifacts/phase-N/`。文件名使用稳定、可读的短横线命名，例如 `weekly-review-w1.md`、`quiz-functions-answers.md`。批改记录应注明日期、表现证据、薄弱点和下一步，不只给分数。
